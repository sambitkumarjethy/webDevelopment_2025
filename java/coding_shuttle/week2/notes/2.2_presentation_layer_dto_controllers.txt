1) AUTO-CONFIGURATION
---------------------
- Spring Boot automatically configures your application based on dependencies in the classpath.
- Removes the need for boilerplate XML configurations.
- Uses @EnableAutoConfiguration (usually via @SpringBootApplication).
- Example:
    If "spring-boot-starter-web" is present â†’ Spring auto-configures Tomcat + DispatcherServlet.

Key Annotations:
- @SpringBootApplication = @Configuration + @EnableAutoConfiguration + @ComponentScan

Benefits:
- Reduces manual setup
- Fast development
- Convention over configuration

---

2) INTERNAL WORKING OF SPRING BOOT
----------------------------------
Step 1: Main class has @SpringBootApplication
Step 2: SpringApplication.run() â†’ bootstraps application
Step 3: Creates ApplicationContext
Step 4: Auto-Configuration kicks in
Step 5: Embedded server (Tomcat/Jetty) starts automatically
Step 6: DispatcherServlet handles requests (for web apps)

6) DYNAMIC URL HANDLING
-----------------------
@PATHVARIABLE:
---------------
Example: /employees/123
- Extracts value from URL path.
- Used when parameter is part of resource identity.

@REQUESTPARAM:
---------------
Example: /employees?id=123
- Extracts value from query string.
- Commonly used for filtering, sorting, optional params.

---

- Example: JSON/XML sent by client â†’ mapped to POJO automatically.
